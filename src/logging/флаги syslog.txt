Работа с syslog в Linux (C++)
1. Основные функции и их параметры
openlog(const char *ident, int logopt, int facility)
ident — префикс для всех сообщений (обычно имя программы).

logopt — флаги поведения (см. раздел 2).

facility — категория источника сообщения (определяет, куда пойдёт лог).

syslog(int priority, const char *message, ...)
priority — уровень важности + категория (комбинация facility из openlog и уровня).

message — строка формата (как в printf), может содержать %m (подставляет strerror(errno)).

closelog(void)
Закрывает соединение с syslog.

2. Флаги и уровни
Флаги для logopt (можно комбинировать через |)
LOG_PID — добавлять PID процесса к каждому сообщению.

LOG_CONS — если syslog недоступен, выводить на консоль.

LOG_NDELAY — немедленно открывать соединение с syslog (не ждать первого сообщения).

LOG_ODELAY — откладывать открытие до первого вызова syslog (по умолчанию).

Категории (facility)
LOG_USER — пользовательские программы (по умолчанию).

LOG_LOCAL0 – LOG_LOCAL7 — зарезервировано для локальных приложений.

LOG_DAEMON — системные службы.

LOG_AUTH — события аутентификации.

LOG_SYSLOG — сообщения от самого syslog.

Уровни важности (priority)
LOG_EMERG — система неуправляема.

LOG_ALERT — требуется немедленное действие.

LOG_CRIT — критические условия.

LOG_ERR — ошибки.

LOG_WARNING — предупреждения.

LOG_NOTICE — нормальные, но значимые условия.

LOG_INFO — информационная рассылка.

LOG_DEBUG — отладочная информация.

# AsyncNet HTTP‑сервер

**Высокопроизводительный асинхронный HTTP‑сервер**, реализованный на C++ с использованием механизма `epoll` для эффективного управления множеством одновременных соединений.

## Ключевые особенности


- **Асинхронная обработка**  
  Использование `epoll` позволяет обслуживать тысячи соединений без создания отдельных потоков.

- **Низкие накладные расходы**  
  Минимальное потребление ресурсов при высокой нагрузке.

- **Гибкая система маршрутизации**  
  Легко расширяемый механизм обработки URI.

- **Встроенная статистика**  
  Учёт общего и текущего числа подключений.

- **Безопасное завершение работы**  
  Обработка сигналов `SIGINT` и `SIGTERM`.

- **Детальное логирование**  
  Запись событий разных уровней важности.

## Функциональность


Сервер поддерживает следующие эндпоинты:

- **`GET /time`**  
  Возвращает текущее время в формате `YYYY‑MM‑DD HH:MM:SS`.

- **`GET /stats`**  
  Отображает статистику подключений:
  - общее число подключившихся клиентов;
  - число активных подключений в данный момент.

- **`GET /shutdown`**  
  Инициирует корректное завершение работы сервера.

- **Любой другой URI**  
  Возвращает страницу с ошибкой `404 Not Found` и списком доступных эндпоинтов.


## Архитектура

Проект построен по модульному принципу.

### 1. Ядро сервера (`http_server`)

- `BaseSession` — базовый класс для обработки сессий;
- `Session<T>` — шаблонный класс сессии с пользовательским обработчиком;
- `Listener<T>` — основной класс сервера, управляющий соединениями;
- `ServerHttp()` — функция‑фабрика для запуска сервера.


### 2. Обработчики запросов (`handler`)

- `RequestHandler` — класс, реализующий логику обработки запросов.


### 3. Вспомогательные модули

- `logger` — система логирования (singleton);
- `utils` — служебные функции и структуры данных;
- `endpoints` — константы для стандартных URI.

### 4. Тестирование (`tests`)
- Набор тестов для проверки функциональности.
